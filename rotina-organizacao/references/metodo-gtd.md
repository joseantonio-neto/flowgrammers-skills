# Método GTD — Getting Things Done Adaptado ao BR

> Baseado em "A Arte de Fazer Acontecer" de David Allen (Editora Sextante, BR).
> Última atualização: Mai/2026. Fontes: Asana PT, Trello BR, Treasy, Witix.

---

## O que é GTD?

GTD (Getting Things Done) é um sistema de gestão de tarefas criado por David Allen em 2001. A premissa central: **o cérebro foi feito para ter ideias, não para guardá-las**.

O método elimina a carga cognitiva de "lembrar de coisas" ao externalizar tudo em um sistema confiável, liberando atenção total para a execução.

**Por que funciona:** Toda "coisa aberta" na mente consome energia. Uma tarefa não registrada cria um loop aberto que drena foco — mesmo quando não está sendo executada.

## Os 5 Passos do GTD

### Passo 1: CAPTURAR (Capture)
Registre absolutamente tudo que vem à mente, sem filtro ou julgamento.

**O que capturar:**
- Tarefas e compromissos
- Ideias e projetos
- Emails para responder
- Listas de compras
- Preocupações e pensamentos recorrentes

**Ferramentas de captura:**
- Bloco de notas físico (sempre à mão)
- App de notas rápidas: Notion, Apple Notes, Google Keep
- Caixa de entrada do email como inbox
- Gravador de voz para quando estiver no trânsito

**Regra:** Só existe um inbox por vez. Muitas caixas criam caos.

---

### Passo 2: ESCLARECER (Clarify / Process)
Processe cada item capturado com uma pergunta central: **"Qual é o próximo passo físico?"**

**Árvore de decisão:**

```
É acionável?
├── NÃO → Lixo / Arquivo / Algum dia/Talvez
└── SIM → Qual é a próxima ação?
    ├── Leva < 2 minutos? → FAÇA AGORA
    ├── Apenas uma ação? → Lista de Próximas Ações
    └── Múltiplas ações? → É um PROJETO → Defina o próximo passo
```

**Regra dos 2 minutos:** Se a ação leva menos de 2 minutos, execute na hora. Não registre.

---

### Passo 3: ORGANIZAR (Organize)
Coloque cada item no lugar certo.

**Categorias de organização:**

| Lista | O que vai aqui |
|---|---|
| Próximas Ações | Tarefas com contexto definido (ex: @Telefone, @Computador, @Casa) |
| Projetos | Qualquer resultado que exige mais de uma ação |
| Aguardando | Coisas que você delegou e precisa acompanhar |
| Algum dia/Talvez | Ideias que não têm prazo ou prioridade definida |
| Calendário | Apenas o que TEM data/hora fixas |
| Arquivo | Referências e informações sem ação |

**Contextos BR comuns:**
- @Email — tarefas que exigem email
- @Ligações — telefonemas e videochamadas
- @Computador — trabalho que exige tela
- @Errands — saídas físicas (banco, farmácia)
- @Conversar com [Nome] — pauta para a próxima reunião

---

### Passo 4: REFLETIR (Reflect)
Revisão sistemática garante que o sistema permaneça confiável.

**Revisão Diária (5-10 min, manhã):**
- Checar calendário do dia
- Revisar lista de próximas ações
- Definir as 3 prioridades do dia (MIT — Most Important Tasks)

**Revisão Semanal (30-60 min, sexta ou segunda):**
Este é o coração do GTD. Sem ela, o sistema morre em 2 semanas.

Checklist da Revisão Semanal:
- [ ] Esvaziar todas as caixas de entrada (email, notas, mente)
- [ ] Revisar e atualizar lista de Projetos
- [ ] Revisar lista de Próximas Ações
- [ ] Revisar lista de Aguardando
- [ ] Revisar lista de Algum dia/Talvez
- [ ] Revisar calendário das próximas 2 semanas
- [ ] Decidir o que vai entrar na semana seguinte

---

### Passo 5: ENGAJAR (Engage)
Execute com confiança total, sabendo que nada importante está escapando.

**Critérios para escolher o que fazer agora (modelo 4 critérios):**
1. **Contexto:** Onde estou e o que tenho disponível (ferramentas, pessoas)?
2. **Energia:** Qual é meu nível de energia agora?
3. **Tempo:** Quanto tempo disponível tenho?
4. **Prioridade:** Dado os anteriores, qual ação tem maior impacto?

## Implementação Prática BR — Primeiros 7 Dias

| Dia | Ação |
|---|---|
| 1 | Escolha UMA ferramenta de captura e configure |
| 2 | Faça o "Mind Sweep" — esvazie tudo da cabeça para o papel |
| 3 | Processe tudo: aplique a árvore de decisão a cada item |
| 4 | Organize nas listas (Projetos, Próximas Ações, etc.) |
| 5 | Configure seu sistema digital (Notion, Todoist, etc.) |
| 6 | Execute seguindo as listas pelo primeiro dia completo |
| 7 | Faça sua primeira Revisão Semanal |

## Erros Comuns e Como Evitar

| Erro | Solução |
|---|---|
| Muitas caixas de entrada | Consolide em 1-3 inboxes máximo |
| Esquecer a revisão semanal | Bloqueie 1h no calendário, todo sábado |
| Listar projetos como tarefas | Projeto = resultado, tarefa = ação física. Separe |
| Deixar itens vagos | "Cuidar do carro" → "Ligar para oficina XPTO" |
| Sistema excessivamente complexo | Comece simples. Adicione complexidade só se precisar |
